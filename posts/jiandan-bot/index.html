Jiandan Bot:一个煎蛋爬虫 + telegram bot

2019.08.31

  前段时间突然喜欢无聊的时候去煎蛋看看无聊图，可是每天打开浏览器看煎蛋太麻烦(，并且 telegram 现存的一个煎蛋爬虫 channel 只推送热门图片，于是就萌生了写一个爬虫，将所有无聊图发送到我 telegram 的想法。

  目前已经写出了基本满足我需求的稳定版本，Github 链接在此

主要结构与内容

├── bot
│   ├── bot.go
│   ├── router.go
│   └── send.go
├── channel
│   └── channel.go
├── config
│   └── config.json
├── crawler
│   └── jiandan.go
├── Dockerfile
├── go.mod
├── go.sum
├── main.go	
├── maker
│   └── jiandan.go
└── types
    ├── config.go
    └── types.go
  • bot 目录
    • bot.go 主要为bot的初始化设置
    • router.go 为bot接收到命令之后的处理路由
    • send.go 为各种消息的send函数

  • channel 目录

    • channel.go 声明了几个全局 channel，用于供各个模块之间互相传递信息，之前是通过传参的方式将 channel 传到使用的地方，后来发现传参过于繁琐，于是采用全局channel的方式

  • crawler 目录

    • jiandan.go 为煎蛋的爬虫，之前的版本为爬取网页，后来抓取到了 api，于是采用直接请求 api 的方式，(顺带吐槽一下煎蛋的 api，好多明明是纯数字的字段却是string，导致接过来之后还要转换)

  • maker 目录

    • jiandan.go 爬虫将爬到的 json 格式的数据通过 channel 传递给 maker，maker将其处理为telegram bot的格式

配置文件格式如下

    {
    "Token": "YOUR_TOKEN",
    "AdminID": 000000,
    "ChannelID": 000000,
    "ChannelUsername": "@YOUTR_CHANNEL_NAME",
    "ApiAddress": "jandan.net_api"
    }

开发过程中的坑

  最先遇到的问题是在 telebot 和 telegram-bot-api 这两个 telegram bot 库中的选择，telebot 的逻辑更清晰一些，感觉写起来比较顺手，telegram-bot-api 用起来其实不怎么顺手，但是 telebot 在标记消息格式的处理上有些问题，没办法正确的标记 markdown 和 html，简单粗暴的修了一下之后提了 pr，过了挺久作者也没有回复，只好转到开发者更活跃的 telegram-bot-api 这个库，虽然写起来有些地方没有那么好用，但是功能实现都很好。

  然后遇到的坑就是想用 telegram 的 webhook 方式，但是反复试了几次之后都有各种各样的问题没能解决，想了下 这个 bot 主要是发送内容比较多，不怎么接收内容，所以放弃了 webhook 的方式，等以后搞懂了再把这个坑填上好了……

  项目是使用 docker 部署的，一开始的 Dockerfile 不知道怎么写，单纯的编译之后运行，发现问题很多，一是镜像太大，完全没有体现出 golang 静态编译的优势，二是不能请求 https 的接口，google 一番之后找到了这篇博客，作者很完善的讲解了各个步骤，根据他给出的 Dockerfile 稍加修改，才部署成功。另外一个坑就是一开始没有注意到，把配置文件也打包进去了，导致修改配置文件之后不起作用，后来才意识到了问题，改为将配置文件挂载进去。

一些想法

  开发之前感觉问题很好解决，但是真正写起来之后才发现中间的坑很多，并且想写成方便以后加入其他爬虫模块的方式，所以在架构上也思考了好久。有些欠缺的东西平时没办法发现，只有在踩到坑的时候才发现自己的知识是多么贫瘠。以后还是多开一些脑洞，写一些练手的东西，不然再怎么学习也只是纸上谈兵了。
